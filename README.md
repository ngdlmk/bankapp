You can start developing by editing the files inside the **app** directory. This project uses [file-based routing](https://docs.expo.dev/router/introduction).

ðŸ“± Screen Features & Functionality
1. Splash Screen (screens/splash)
- Animation: Uses Animated.parallel for a smooth Scale & Fade-in effect on the logo.

- Logic: Checks SecureStore for an existing user token.

- Routing: Automatically directs to Login (if new) or Security Screen (if returning).

2. Login Screen (screens/login)
- Auth0 Integration: Embeds a secure WebView to handle OAuth 2.0 authentication flow.

- Token Management: Intercepts the authentication code/token and securely saves it to the device Keychain.

3. Security Screen (screens/security)
- Context Aware: Acts as both a "Create PIN" screen (first setup) and "Unlock" screen.

- Biometrics: Automatically triggers FaceID/TouchID via expo-local-authentication.

- PIN Fallback: Custom-built numeric keypad with dot indicators for users without biometrics.

- Privacy: This screen is automatically invoked when the app moves to the background to protect sensitive data.

4. Dashboard (screens/dashboard)
- Dynamic Theming: Supports Light/Dark mode switching (via Unistyles).

- Data Visualization: Lists accounts with formatted currency.

- Interactivity: Tapping an account opens a modal with transaction history.

- Smart Styling: Income (Credit) and Expense (Debit) transactions are automatically colored green or black based on value.

5. Help / AI Support (screens/help)
- ChatGPT Integration: Connects to OpenAI API to answer banking queries.

- Persistence: Uses MMKV to store chat history locally, so messages aren't lost on close.

- Context: Sends previous conversation history to OpenAI to maintain context.
